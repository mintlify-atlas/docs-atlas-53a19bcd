Catalyst is designed to build itself. Once you have a bootstrap binary, you use Catalyst to compile subsequent versions of Catalyst.

## Building for development

With a working `catalyst` binary on your `PATH`, build a debug version:

```bash
catalyst build --profiles debug
```

Run the test suite:

```bash
catalyst test
```

Format code before committing:

```bash
catalyst fmt
```

## Project structure

- `src/` — Implementation files
- `include/` — Public header files
- `src/subcommands/` — One directory per `catalyst` subcommand
- `docs/` — Documentation source
- `tests/` — Unit and integration tests

See the [contributing guide](/contributing) for code style requirements and PR guidelines.
