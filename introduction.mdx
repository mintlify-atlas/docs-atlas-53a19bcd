Catalyst is a declarative meta build system designed to modernize C and C++ workflows. Instead of wrestling with complex CMake scripts or fragile Makefiles, you describe your project in a simple YAML file and let Catalyst handle compilation, dependency management, and build orchestration.

## Why Catalyst?

C and C++ builds have historically required learning a separate scripting language — CMake's DSL, Makefile syntax, or Meson's Python-like dialect. Catalyst replaces that with a single, readable YAML file that any developer can understand at a glance.

## Quick look

A minimal `catalyst.yaml` that builds a C++20 binary and pulls in the `fmt` library:

```yaml catalyst.yaml
manifest:
  name: my-app
  type: BINARY
  version: 0.1.0
  tooling:
    CXX: clang++
    CXXFLAGS: -std=c++20 -Wall -Wextra
  dirs:
    include: [include]
    source: [src]
    build: build

dependencies:
  - name: fmt
    source: git
    url: https://github.com/fmtlib/fmt.git
    version: 10.0.0
```

Then build and run:

```bash
catalyst build
catalyst run
```

## Core commands

| Command | Description |
|---|---|
| `catalyst init` | Create a new project |
| `catalyst build` | Build the project |
| `catalyst run` | Run the built binary |
| `catalyst test` | Run project tests |
| `catalyst add` | Add a dependency |
| `catalyst lock` | Pin dependency versions |
