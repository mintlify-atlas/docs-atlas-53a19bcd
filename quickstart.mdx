This guide walks you through creating a new C++ project from scratch, configuring it with a `catalyst.yaml` file, building the binary, running it, and pulling in an external dependency.

## Create and run your first project

<Steps>
  <Step title="Initialize the project">
    Create a directory for your project and run `catalyst init` inside it:

    ```bash
    mkdir my-app
    cd my-app
    catalyst init
    ```

    Catalyst generates the following project structure:

    ```
    my-app/
    ├── catalyst.yaml
    ├── src/
    │   └── my-app.cpp
    └── include/
    ```

    - `catalyst.yaml` — the project configuration file
    - `src/my-app.cpp` — a default entry point with a Hello World program
    - `include/` — directory for header files
  </Step>

  <Step title="Review the configuration">
    Open `catalyst.yaml`. A typical starting configuration looks like this:

    ```yaml catalyst.yaml
    manifest:
      name: my-app
      type: BINARY
      version: 0.1.0
      tooling:
        CXX: clang++
        CXXFLAGS: -std=c++20 -Wall -Wextra
      dirs:
        include: [include]
        source: [src]
        build: build
    ```

    `type: BINARY` tells Catalyst to produce an executable. Change this to `STATICLIB` or `SHAREDLIB` to build a library instead. The `tooling` block sets the compiler and flags — swap `clang++` for `g++` if you are using GCC.
  </Step>

  <Step title="Build the project">
    Run the build command from your project root:

    ```bash
    catalyst build
    ```

    Catalyst performs three steps:

    1. **Generate** — produces build files using CBE by default (or Ninja / GNU Make if configured)
    2. **Compile** — compiles all source files under `src/`
    3. **Link** — links the compiled objects into the output binary

    The final binary is written to `build/my-app`.
  </Step>

  <Step title="Run the application">
    Execute the binary with:

    ```bash
    catalyst run
    ```

    You should see:

    ```
    Hello, Catalyst!
    ```
  </Step>
</Steps>

## Adding a dependency

Use `catalyst add` to pull in an external library. The command accepts a source type (`git`, `vcpkg`, or `system`) and optional flags for version pinning.

To add `fmt` from GitHub at version 10.0.0:

```bash
catalyst add git https://github.com/fmtlib/fmt.git -v 10.0.0
```

Catalyst appends the dependency to your `catalyst.yaml` automatically:

```yaml catalyst.yaml
dependencies:
  - name: fmt
    source: git
    url: https://github.com/fmtlib/fmt.git
    version: 10.0.0
```

Run `catalyst build` again to fetch and link the library.

<Tip>
  Pin your dependencies to a specific version and commit the resulting lock file to ensure reproducible builds across machines.
</Tip>
