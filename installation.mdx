---
title: "Build and install Catalyst from source"
description: "Build Catalyst from source using Clang 16+ or GCC 13+ and CMake 3.20+. This guide covers prerequisites, the bootstrap build process, and verifying your install."
---

Catalyst is currently distributed as source only. You build it in two stages: first compile a bootstrap binary using CMake, then use that bootstrap binary to build the full Catalyst toolchain.

<Warning>
  Catalyst is under active development. Expect breaking changes and rough edges as the project evolves toward a stable release.
</Warning>

## Prerequisites

Make sure the following tools are available on your system before you begin.

| Tool | Minimum version |
|---|---|
| CMake | 3.20 |
| Clang | 16+ (must support C++23) |
| GCC | 13+ (must support C++23) |
| Git | Any recent version |

You need either Clang or GCC — you do not need both. Clang is recommended. You also need a build backend: CBE (recommended), Ninja, or GNU Make.

## Build from source

<Steps>
  <Step title="Clone the repository">
    Clone the Catalyst repository from GitHub:

    ```bash
    git clone https://github.com/CatalystCPP/catalyst-build-system.git
    cd catalyst-build-system
    ```
  </Step>

  <Step title="Build the bootstrap binary">
    Check out the stable 1.0.0 tag and build the bootstrap version of Catalyst using CMake. This produces a minimal binary capable of building Catalyst itself.

    ```bash
    git checkout tags/1.0.0
    cmake -B build-bootstrap -S . -DCMAKE_BUILD_TYPE=Release
    cmake --build build-bootstrap --config Release
    ```

    The bootstrap binary is written to `build-bootstrap/catalyst`.
  </Step>

  <Step title="Build Catalyst">
    Switch to the development branch and use the bootstrap binary to build the full Catalyst toolchain:

    ```bash
    git checkout dev-1.1.0
    mkdir build
    ./build-bootstrap/catalyst build
    ```

    Build artifacts are written to the `build/` directory.
  </Step>

  <Step title="Verify the installation">
    Confirm the build succeeded by checking the version:

    ```bash
    build/catalyst -v
    ```

    You should see:

    ```
    1.1.0
    ```

    Add `build/` to your `PATH` to run `catalyst` from any directory.
  </Step>
</Steps>

<Note>
  Catalyst is early-stage software. If you encounter unexpected behavior or build failures, please open an issue on [GitHub](https://github.com/CatalystCPP/catalyst-build-system).
</Note>
